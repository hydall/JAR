# Janitor AI Ripper

JAR (Janitor AI Ripper) — извлекает лорбуки и персонажей из JanitorAI, как публичные, так и приватные, локально на вашем ПК и в автоматическом режиме.

## Требования

- **ПК с Windows / Linux / Mac и графическим интерфейсом.** JAR запускает полноценное окно браузера, поэтому без GUI работать не будет.
- JAR **не работает на Termux** из коробки, и **поддержка Termux не предоставляется.**
- Для извлечения с мобильных устройств попробуйте [GlazeFlutter](https://github.com/hydall/GlazeFlutter) — разрабатываемый фронтенд для LLM-ролевых игр. Инструменты JAR встроены в него по умолчанию.

## Установка и запуск

### Шаг 1. Получите исходный код

Вариант А — без Git: откройте https://github.com/hydall/JAR, нажмите зелёную кнопку **Code → Download ZIP**, распакуйте архив и перейдите в полученную папку.

Вариант Б — с Git:
```bash
git clone https://github.com/hydall/JAR.git
cd JAR
```

### Шаг 2. Запустите приложение

**Windows (проще всего):** дважды кликните **`start.bat`**.
При первом запуске он установит Node.js, зависимости и Chromium, затем запустит приложение. Последующие запуски просто стартуют.

**Linux / Mac (или ручной запуск на Windows):**

1. Установите **Node.js LTS** с https://nodejs.org/en/download (если ещё не установлен).
2. Из папки проекта установите зависимости и браузер Chromium:
   ```bash
   npm install
   ```
   (Chromium устанавливается автоматически через `postinstall`.)
3. Запустите приложение:
   ```bash
   npm start
   ```

После запуска приложение откроется в вашем браузере по умолчанию. **Не закрывайте окно терминала / консоли во время работы** (остановка — `Ctrl+C`).

### Шаг 3. Настройте LLM для извлечения

Откройте **⚙ Настройки**, чтобы настроить эндпоинт извлечения — любой OpenAI-совместимый эндпоинт (OpenRouter, OpenAI, локальный), который будет использоваться для построения лорбука в JSON (ключи и правила активации).

### Шаг 4. Войдите в JanitorAI

Приложение использует **отдельный браузер** (папка `user-data/`) — вы входите в JanitorAI один раз внутри него, и сессия сохраняется. Всё извлечение происходит **под вашей учётной записью**; инструмент отправляет текст персонажа как сообщения чата для активации записей лорбука. **Рекомендуется использовать одноразовую учётную запись.**

Программа не собирает и не хранит ваши учётные данные — она только читает куки сессии JanitorAI для проверки авторизации.

## Использование

1. **Вход** — нажмите кнопку **🔓** в шапке. Откроется отдельное окно браузера на janitorai.com/login. Войдите как обычно. Приложение автоматически обнаружит сессию.

2. **Просмотр персонажа** — вставьте URL персонажа в боковую панель и нажмите **извлечь** (extract). Это загрузит карточку, аватар и все публичные лорбуки.

3. **Извлечение приватной карточки** — если описание персонажа скрыто, нажмите **извлечь карточку** (extract card) на вкладке *карточка персонажа*. Инструмент создаст чат, отправит пробное сообщение и перехватит карточку из отправленного запроса.

4. **Извлечение закрытого лорбука** — нажмите **извлечь** (extract) на вкладке *лорбук*. Отправляет выбранный контекст для активации записей лорбука на серверах Janitor. Затем эти записи перехватываются. Нажмите «построить лорбук» (build lorebook) и выберите контекст, чтобы отправить записи LLM на ваш выбор для реконструкции лорбука.

5. **Скачивание** — нажмите **скачать .json** (download .json) для файла World Info для SillyTavern. Импортируйте через World Info → Import. Карточки персонажей скачиваются в PNG или JSON на вкладке *карточка персонажа*.

## Как это работает

JanitorAI использует Cloudflare для защиты от автоматизации. JAR запускает **полноценное окно браузера**, которое проходит через Cloudflare и выполняет все запросы оттуда.

**Извлечение приватной карточки персонажа** — JAR создаёт фиктивный пресет прокси, отправляет сообщение в чат и перехватывает ответ `generateAlpha`: собранный промпт, содержащий карточку персонажа, обёрнутую в теги.

**Извлечение приватного лорбука** — закрытые лорбуки обрабатываются на сервере JanitorAI. Полное извлечение в исходном виде невозможно. В качестве обходного пути JAR отправляет в чат карточку персонажа, описание из каталога и первое сообщение. Этот текст достигает сервера JanitorAI, где активирует записи лорбука. Сервер вставляет эти записи в промпт. JAR перехватывает ответ `generateAlpha` и изолирует записи лорбука, отбрасывая всё остальное.

**Публичные лорбуки и персонажи** — они публичны и загружаются напрямую с эндпоинта `/hampter/characters`. Если у персонажа есть и публичные, и закрытые лорбуки, текст публичных записей автоматически вычитается из перехваченного промпта перед извлечением, оставляя только закрытые записи.

После извлечения вы можете:
- Скачать **сырой лорбук** без ключей и правил.
- Отправить записи выбранной LLM (вместе с выбранным контекстом), чтобы она построила полноценный лорбук.

Если персонаж использует общий лорбук (например, вселенной или общий лорбук), этот метод может не активировать все записи автоматически. Единственный способ извлечь их — вручную вводить ключевые слова-триггеры во время сбора записей. Затем эти ключи можно отправить LLM при построении лорбука для дополнительного контекста.
